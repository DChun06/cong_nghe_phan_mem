\section{Lựa Chọn Mô Hình Kiến Trúc Hệ Thống}
Nhằm mục đích đạt được tính độc lập cao giữa các khối chức năng và tăng cường khả năng kiểm thử (Testability) theo nguyên lý chương 9-13 của Pressman, hệ thống Backend được triển khai theo mô hình kiến trúc phân lớp \textbf{3-Layer Architecture} bao gồm: Tầng giao tiếp API (Presentation Layer), Tầng xử lý nghiệp vụ cốt lõi (Business Logic Layer), và Tầng tương tác truy cập cơ sở dữ liệu (Data Access Layer).

\section{Mã Nguồn Khung Triển Khai Hệ Thống API bằng Python và FastAPI}
Dưới đây là mã nguồn cài đặt khung chạy hoàn chỉnh cho Backend, tích hợp cấu trúc tiếp nhận dữ liệu sinh viên và xử lý logic phân tích khoảng cách năng lực:

\begin{lstlisting}[language=Python, caption=Tệp mã nguồn chính main.py xử lý API Backend]

	from pydantic import BaseModel
	from typing import List
	
	app = FastAPI(

	student_id: int
	full_name: str
	skills: List[str]
	target_job: str

	return {"status": "Online", "message": "Career Orientation API Core Framework"}

	def generate_roadmap(profile: StudentProfileSchema):
	"""
	Business Logic Layer: Hiện thực hóa logic tính toán khoảng cách năng lực 
	giữa hồ sơ thực tế của sinh viên và bộ khung chuẩn ngành nghề.
	"""
	try:
	student_skills_set = set(profile.skills)
	
	# Mô phỏng bộ khung năng lực chuẩn từ Database cho vị trí Backend Engineer
	master_framework_skills = {"Python", "FastAPI", "SQL Server", "Docker", "Clean Architecture", "Git"}
	
	# Thực hiện thuật toán tìm tập hợp kỹ năng còn thiếu (Skill Gap Analysis)
	missing_skills = list(master_framework_skills.difference(student_skills_set))
	
	return {
		"student_id": profile.student_id,
		"full_name": profile.full_name,

		"detected_skill_gap": missing_skills,
		"recommended_modules_count": len(missing_skills),
		"status": "Process_Success"
	}
	except Exception as e:
	raise HTTPException(status_code=500, detail=f"Lỗi hệ thống xử lý nội bộ: {str(e)}")
