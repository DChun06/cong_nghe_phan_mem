\section{Overview}

\subsection{Project Scope and Estimation}

The \textbf{Personalized Career Orientation \& Learning Roadmap Platform for Software Engineering Students} is a web-based system designed to assist Software Engineering students in identifying suitable career paths and generating personalized learning roadmaps based on their academic performance, technical skills, interests, and career goals.

\textbf{Total Estimated Effort: 152 Man-days}

The objectives of the project are as follows.

\begin{itemize}
\item Develop a web-based personalized career orientation platform.
\item Recommend suitable Software Engineering career paths.
\item Generate personalized learning roadmaps.
\item Track students' learning progress.
\item Recommend appropriate learning resources.
\item Improve students' career planning capability.
\item Complete the project within one academic semester.
\end{itemize}

\section{Management Approach}

\subsection{Development Methodology}

The project adopts the Agile Scrum methodology.

The reasons include:

\begin{itemize}
\item Continuous customer feedback
\item Flexible requirement changes
\item Incremental software delivery
\item Continuous testing
\item Better project monitoring
\end{itemize}

Each sprint lasts two weeks.

\subsection{Quality Management}

Quality assurance activities include:

\begin{itemize}
\item Requirement Review
\item Code Review
\item Unit Testing
\item Integration Testing
\item System Testing
\item User Acceptance Testing
\end{itemize}

\section{Configuration Management}

\subsection{Document Management}

The project documentation is maintained using:

\begin{itemize}
\item Microsoft Word
\item Microsoft Excel
\item Google Drive
\end{itemize}

\subsection{Source Code Management}

The source code is managed using GitHub and Git.

The following branching strategy is adopted:

\begin{itemize}
\item main
\item develop
\item feature/*
\item release/*
\item hotfix/*
\end{itemize}

\subsection{Version Control Policy}

\begin{itemize}
\item Every feature is developed in an independent branch.
\item Pull Requests are mandatory before merging.
\item Code review is required before approval.
\item Semantic Versioning is applied.
\end{itemize}

Example versions:

\begin{itemize}
\item Version 1.0.0
\item Version 1.1.0
\item Version 2.0.0
\end{itemize}
