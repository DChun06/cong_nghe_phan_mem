\subsection{Installation Instructions}

\subsubsection{Option 1 — Access the Online Demo}

The fastest way to explore Release 1.0 is through the hosted GitHub Pages demo, which requires no installation:

\begin{itemize}[nosep]
    \item Step 1: Open a web browser.
    \item Step 2: Navigate to \url{https://software-engineering-students.onrender.com/}.
    \item Step 3: The landing page of \textit{SE Career Compass} will load directly; no login credentials are required to explore the demo flows.
\end{itemize}

\subsubsection{Option 2 — Run the Project Locally}

Developers or evaluators who want to run and modify the source code can set up the project locally by following these steps:

\begin{itemize}[nosep]
    \item Step 1: Download \& install \textbf{Git}: \url{https://git-scm.com/}.
    \item Step 2: Download \& install \textbf{Node.js} (v18 or higher): \url{https://nodejs.org/}.
    \item Step 3: Download \& install \textbf{Visual Studio Code}: \url{https://code.visualstudio.com/}.
    \item Step 4: Open a terminal and clone the repository:
\begin{verbatim}
git clone https://github.com/phucvobao/Software-Engineering-Students.git
\end{verbatim}
    \item Step 5: Navigate into the project folder:
\begin{verbatim}
cd Software-Engineering-Students
\end{verbatim}
    \item Step 6: If the project uses npm dependencies, install them:
\begin{verbatim}
npm install
\end{verbatim}
    \item Step 7: Serve the site locally (for a static site, either open \texttt{index.html} directly in a browser or run a local server, e.g.):
\begin{verbatim}
npx serve .
\end{verbatim}
    \item Step 8: Open the address shown in the terminal (e.g. \texttt{http://localhost:3000}) in a web browser to view the application.
\end{itemize}

\subsubsection{Deployment}

The demo is deployed automatically to \textbf{GitHub Pages} whenever changes are pushed to the repository's publishing branch, so the public URL always reflects the latest committed version of Release 1.0.

\section{User Manual}

\subsection{Overview}

The platform serves four internal actors, each with a dedicated working area, as summarized below.

\textbf{Student}
\begin{itemize}[nosep]
    \item Authentication: Register, Login (email/password or Google OAuth), Forgot Password, Logout.
    \item Profile: Manage profile, upload avatar, upload transcript, upload CV, link GitHub account.
    \item Career Orientation: Chat with the AI Virtual Mentor, view recommended career roles, compare career paths, view required skills.
    \item Learning Roadmap: Generate and view the personalized learning roadmap (skill tree), update progress, receive learning reminders.
    \item Skill Gap Analysis: View skill gap report and export it as a PDF.
    \item Market Pulse: View labor-market trends, technology trends, salary information, and job recommendations.
    \item E-Portfolio: Sync GitHub repositories, generate an e-portfolio, and share it via a public link.
\end{itemize}

\textbf{Academic Counselor}
\begin{itemize}[nosep]
    \item Login to the counselor workspace; view student profiles and skill gap reports; recommend career paths and courses; track student progress; comment on learning plans; schedule and review consultations.
\end{itemize}

\textbf{Industry Mentor}
\begin{itemize}[nosep]
    \item Login to the mentor workspace; view student e-portfolios and projects; recommend technologies and provide technical advice; evaluate progress; approve portfolios and suggest improvements.
\end{itemize}

\textbf{Administrator}
\begin{itemize}[nosep]
    \item Manage user accounts, career roles, the skill graph, courses, learning resources, job-trend data, AI configuration, and system reports.
\end{itemize}
